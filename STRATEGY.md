# Стратегия разработки проекта

## Обзор

Проект представляет собой кроссплатформенное десктопное приложение для генерации документов и коммерческих предложений.

## Технологический стек (утвержден)

✅ **Tauri** (Rust) - фреймворк для десктопных приложений
- Поддержка React и Svelte ✅
- Кроссплатформенность (Linux, Windows, macOS) ✅
- Малый размер бандла
- Высокая производительность

✅ **Python** - для генерации документов
- python-docx для .docx
- reportlab для PDF
- jinja2 для шаблонизации

✅ **Frontend**: React или Svelte (решение принято - на выбор разработчика)

## Архитектурные решения

### 1. Интеграция Rust-Python

**Выбрано для MVP**: Subprocess подход
- Простота реализации
- Гибкость (можно менять Python код без перекомпиляции)
- Достаточная производительность для MVP

**Для production**: Рассмотреть PyO3 для лучшей производительности

### 2. Хранение данных

**Выбрано**: SQLite (через Tauri plugin sql)
- Структурированные данные
- Запросы и фильтрация
- Локальное хранение

JSON файлы для конфигурации и простых данных

### 3. Frontend Framework

**Варианты**:
- **Svelte** (рекомендуется) - легковесно, быстро, просто
- **React** - больше готовых компонентов

**Решение**: Svelte для MVP (можно позже перейти на React если нужны специфичные библиотеки)

## Риски и митигация

| Риск | Вероятность | Митигация |
|------|-------------|-----------|
| Проблемы с интеграцией Rust-Python | Средняя | Начать с Subprocess, есть fallback на PyO3 |
| Производительность генерации | Низкая | Кэширование, асинхронность |
| Кроссплатформенные баги | Средняя | Раннее тестирование на всех платформах |
| Сложность шаблонов | Низкая | Начать с простых, постепенно усложнять |
