## Технические решения

### Выбор метода интеграции Rust-Python

**Вариант 1: PyO3 (рекомендуется)**
- ✅ Прямая интеграция, быстрее
- ✅ Меньше зависимостей
- ❌ Сложнее настройка
- ❌ Нужна компиляция Python в бинарник

**Вариант 2: Subprocess**
- ✅ Проще реализация
- ✅ Гибкость (можно менять Python код без перекомпиляции)
- ❌ Медленнее
- ❌ Нужен установленный Python

**Вариант 3: Локальный HTTP сервер**
- ✅ Полная изоляция
- ✅ Легко тестировать отдельно
- ❌ Сложнее архитектура
- ❌ Больше overhead

**Рекомендация**: Начать с Subprocess для быстрого прототипа, затем перейти на PyO3 для production.

### Хранение данных

**Вариант 1: SQLite (через Tauri plugin)**
- ✅ Структурированные данные
- ✅ Запросы
- ✅ Масштабируемость

**Вариант 2: JSON файлы**
- ✅ Простота
- ✅ Читаемость
- ❌ Медленнее для больших объемов

**Рекомендация**: SQLite для основных данных, JSON для конфигурации.
